Intro - css란?

[ cascading style sheets ]

웹 문서의 전반적인 스타일을 미리 저장해 둔 스타일시트이다. 문서 전체의 일관성을 유지할 수 있고, 세세한 스타일 지정의 필요를 줄어들게 하였다.

HTML을 이용해서 웹 페이지를 제작할 경우 전반적인 틀에서 세세한 글꼴 하나 하나를 일일이 지정해주어야 하지만, 웹 페이지의 스타일(작성형식)을 미리 저장해 두면 웹 페이지의 한 가지 요소만 변경해도 관련되는 전체 페이지의 내용이 한꺼번에 변경되므로, 문서 전체의 일관성을 유지할 수 있고 작업 시간도 단축된다.

[네이버 지식백과] CSS [cascading style sheets] (두산백과)

10/3

오늘부터는 css를 공부한다. 스타일 태그에 대해 배웠다. 스타일을 정의할 태그를 선택하는 선택자selector, 선언/효과declaration, 속성property, 값value 라는 요소들이 있다. 이것들은 html문서 내에 존재할 때에는 head안에 들어간다.

10/4

다양한 선택자
특정 선택자를 지정하여 값을 줄 수 있다. 우선순위는 id > class > tag 순으로 강하다. 가장 포괄적인 태그들을 가지고 전체적인 디자인을 만들고, 예외적인 것들을 id등으로 따로 지정해주는 것이다.
클래스에는 다음과 같이 여러개의 값이 들어갈 수 있지만ㅡclass="saw active"ㅡ 좋은 방법은 아니다. 효율적인 전체 시스템을 바꾸지 않으면서, 정교하게 가변적인 부분들을 짜나가는 것이 중요하다. css selector만 검색해도 많은 선택자들이 어떤 기능을 하며 어떻게 정의하는지 볼 수 있다.

예를 들어 이미 읽은 페이지를 표시하고 싶다고 할 때, 선택자 이름을 saw라고 해보자. 그러면 class="saw"를 지정하고 싶은 요소들에게 붙여준다. 그리고 스타일에서 .saw {} 의 내용을 만들면 된다.
class="active" .active {} id="active" #id {}

박스 만들기
h1, h2, h3등의 제목태그는 일반적으로 페이지 너비 전체를 사용한다. 이것들을 블럭이라고 부른다. 그리고 자신의 컨텐츠 만큼의 크기만 차지하는 것을 인라인이라고 부른다. 보더라인 색깔을 지정해보면 알 수 있다. border-width, border-color, border-style 등을 쓴다. display:block;등으로 블록과 인라인을 바꿀 수 있고, display:none; 으로 숨길 수 있다. 최대한 중복되는 내용을 줄일 수 있게 선택자도 쉼표로 잇고, 속성값고 한줄에 넣어주자.
박스의 여백을 설정할 때는 패딩과 마진을 사용한다. padding:-px;, margin:-px; border를 기준으로 안쪽이 패딩, 바깥쪽이 마진이다. 자세한 것은 구글 이미지검색을 이용하자. 또는 '검사'를 눌러 만들어져있는 페이지가 어떤 css를 사용하고 있는지 보는 것도 좋다.

본격적으로 그리드를 만들어보자. 먼저 영역을 설정하는 데에는 div와 span을 쓴다. 전자는 블록, 스판은 인라인 요소다.

반응형 디자인에 대해 알아보자. 창 크기가 달라지면 창 너비가 바뀌게 만들어야 한다. 이때 사용하는 것이 바로 미디어쿼리라고 하는 코드다. @media 안에 조건을 넣는다. min-width를 넣으면 ~보다 작을때 라는 조건을 거는 셈이다. div의 너비에 대한 조건을 넣고 내비게이션의 위치를 조정하거나 하면 된다.
스타일시트
style에 넣었던 코드들을 모두 style.css파일 안에 붙여넣고, 이제까지 만들었던 html에 연결시키자. link안에 style.css를 넣으면 된다. 이제 모든 html를 일일히 수정하지 않아도 css만 수정할 수 있다. 용량도 줄고, 유지보수가 편리해지고, 공동작업의 효율성도 생긴다. 네트워크 측면에서도, style.css 캐시가 한번 저장되면 트래픽이 더 적어지는 중요한 효과가 생긴다! 즉 돈을 아낄 수 있다.
